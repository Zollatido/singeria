# ☁️ SINGERIA — คู่มือสถาปัตยกรรมระบบสังเคราะห์เสียงร้อง AI บน Google Cloud

เอกสารนี้สรุปแนวทางการสร้างเซิร์ฟเวอร์สำหรับ **Singing Voice Synthesis (SVS)** ของโปรเจกต์ **SINGERIA** ซึ่งเป็นเครื่องมือสังเคราะห์เสียงร้อง AI ประสิทธิภาพสูงในรูปแบบ **Standalone Plugin (VST/AU), SDK และ API** สำหรับนักพัฒนาครับ

---

## 1. การเลือก AI Core Engine (โมเดล Open-Source)

ปัจจุบันมีโมเดล AI ร้องเพลงแบบ Open-Source ที่เสียงสมจริงให้เลือกใช้ 2 ประเภทหลักที่เหมาะกับระบบ **SINGERIA**:

### A. DiffSinger / OpenUtau (แนะนำสำหรับโปรเจกต์นี้)
*   **การทำงาน:** สังเคราะห์เสียงจาก "ศูนย์" โดยรับค่าเป็น **ตัวโน้ต (MIDI) + เนื้อร้อง (Lyrics)**
*   **ข้อดี:** เหมาะกับ SINGERIA มาก เพราะระบบมี Piano Roll (Module B) สำหรับวาดโน้ตและแต่งคำร้องอยู่แล้ว สามารถโยนข้อมูลโน้ตไปให้ AI ร้องออกมาเป็นคำๆ ได้เลย ไม่ต้องมีคนร้องไกด์นำ

### B. RVC (Retrieval-based Voice Conversion) / SO-VITS-SVC
*   **การทำงาน:** เป็นการ "แปลงเสียง" (Voice-to-Voice) คือต้องมีนักร้องจริงร้องไกด์นำไปก่อน แล้ว AI จะเปลี่ยนเนื้อเสียงให้เป็นเสียงเป้าหมาย (หรือ Custom Voice ที่ผู้ใช้โคลนผ่าน **Module A**)
*   **ข้อดี:** เหมือนคนจริงที่สุด มีสไตล์การหายใจและลูกคอติดมาแบบคนร้องไกด์ 100%

## 2. โครงสร้าง 3 โมดูลหลักของ SINGERIA

| โมดูล | ชื่อ | หน้าที่หลัก |
| :---: | :--- | :--- |
| **A** | Voice Extractor | โคลนเสียงจาก Microphone หรือ Audio File ภายใน 15 วินาที |
| **B** | Pro-Expression | ปรับแต่ง Pitch, Vibrato, Dynamics, Expression ด้วย Piano Roll |
| **C** | SVS Engine | รวมทุกอย่างแล้ว Render เสียงร้อง AI ผ่านปุ่ม RENDER SINGERIA |

---

## 3. โครงสร้างฮาร์ดแวร์บน Google Cloud (Infrastructure)

การจำลองเสียง AI ต้องใช้การ์ดจอ (GPU) ในการประมวลผลให้เร็วครับ หากใช้ Google Cloud Platform (GCP) อย่างเดียว จะใช้ Services ดังนี้:

*   **ประมวลผล (Compute):** ใช้ **Cloud Run (รัน GPU)** แบบ Serverless หรือ **Compute Engine (VM)** ติดตั้ง NVIDIA T4 หรือ L4
*   **เก็บข้อมูล (Storage):** ใช้ **Cloud Storage (GCS)** เก็บไฟล์โมเดลเสียง (Custom Voice) และไฟล์เสียง `.wav` ผลลัพธ์
*   **จัดการคิวงาน (Queue):** ใช้ **Cloud Pub/Sub** รับคำสั่ง Render กรณีมีผู้ใช้พร้อมกันหลายคน และ **API Gateway** เช็คสิทธิ์

---

## 4. โฟลว์การทำงานจริงของ API (Workflow)

1.  **Frontend / DAW Host:** ผู้ใช้ปรับแต่งร้องผ่านส่วนควบคุมของ **SINGERIA Plugin/SDK** แล้วกดปุ่ม `"RENDER SINGERIA"`
2.  **Request:** แอปยิง HTTP POST ส่งข้อมูลตัวโน้ต จังหวะ เนื้อร้อง และไฟล์/ไอดีเสียง Custom Voice ไปที่ API
3.  **Processing (บนคลาวด์):**
    *   Python (FastAPI) รับข้อมูลเข้าคิว (Pub/Sub)
    *   สั่งให้ DiffSinger/RVC ประมวลผลลัพธ์ผ่าน GPU
4.  **Response:** เซิร์ฟเวอร์อัปโหลดไฟล์เสียง `.wav` ขึ้น Cloud Storage แล้วส่ง URL กลับ
5.  **Playback:** ระบบ Track ของ SINGERIA โหลดไฟล์ `.wav` มาเล่นซ้อนใน Mixer ทันที

## 5. 💸 การประเมินต้นทุน (Cost Estimation)

> **ตัวอย่างเงื่อนไข:** เพลงความยาว 10 นาที (600 วินาที) ต่อ 1 แทร็ค / 1 ครั้ง

### รูปแบบที่ 1: Serverless / Pay-per-second (แนะนำสำหรับตอนเริ่มต้น)
ระบบคิดเงินเฉพาะ "วินาที" ที่ AI กำลังออกแรง Render เท่านั้น (เช่น Google Cloud Run GPU, Modal.com, RunPod)

*   **เวลาประมวลผล:** AI ทำงานเร็วกว่าเวลาจริง 3-5 เท่า — เพลง 10 นาที ใช้เวลา Render จริงประมาณ **120 วินาที (2 นาที)**
*   **ค่าเช่า GPU:** ตกวินาทีละประมาณ $0.0004 USD
    *   *ต้นทุนการประมวลผล = 120 × $0.0004 = **$0.048 USD***
*   **ค่าส่งข้อมูลไฟล์ (Bandwidth):** ไฟล์ 10 นาที ≈ 100MB (0.1GB) ตก GB ละ $0.10 USD
    *   *ต้นทุนส่งไฟล์ = 0.1 × $0.10 = **$0.01 USD***
*   **สรุปต้นทุนรวม:**
    *   👉 **ประมาณ $0.06 – $0.10 USD (≈ 2–4 บาทไทย) ต่อ 1 เพลงยาว 10 นาที**

### รูปแบบที่ 2: Dedicated Server (เช่า 24 ชม.)
เหมาะกับแอปที่มีคนใช้งานหลักหมื่นคนต่อวัน

*   **สเปค:** 4 vCPU, RAM 15GB + NVIDIA T4 1 ใบ
*   **ค่าเช่ารายเดือน:** ประมาณ **$250 – $350 USD** (≈ 8,500 – 12,000 บาท)
*   **ต้นทุนเฉลี่ยต่อการกด Render 1 ครั้ง (เพลง 10 นาที):**
    *   ถ้าทั้งเดือนมีคนกด 100 ครั้ง = ตกครั้งละ **$3.00 USD** (แพง)
    *   ถ้าทั้งเดือนมีคนฮิตกด 10,000 ครั้ง = ตกครั้งละ **$0.03 USD** (คุ้มค่า)
